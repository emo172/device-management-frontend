# 认证页面去卡片化重构设计

## 背景

当前 4 个公开认证路由已经具备统一的 `auth` 布局与基础交互能力，但视觉结构仍停留在“左侧说明面板 + 右侧实体卡片”的过渡方案：

- `src/layouts/AuthLayout.vue` 负责双栏壳层与背景
- `src/views/auth/Login.vue`、`src/views/auth/Register.vue` 分别承载登录与注册表单
- `src/views/auth/components/PasswordResetPanel.vue` 被忘记密码页与重置密码页复用

本次改造目标不是调整认证逻辑，而是在保留现有路由、表单校验、回跳与验证码流程的前提下，依据已确认的设计方向重做认证页布局与文案表达。实现范围按“4 个公开路由、3 类视觉页”理解：登录页、注册页、密码相关页（`/forgot-password` 与 `/reset-password`）。

## 已确认的设计结论

### 1. 整体视觉方向

- 四个公开路由对应的三类页面统一采用“整张分栏画布”而不是“右侧悬浮卡片”
- 页面内部不再强调卡片包裹感，主要通过留白、分栏、边界线和标题层级建立信息结构
- 保留浅色渐变背景与现有品牌色体系，避免脱离当前控制台视觉基线

### 2. 页面结构方向

- 登录页、注册页与密码相关页都保留双栏结构
- 左栏负责品牌说明、流程价值、语义图标与补充说明
- 右栏负责页面主操作，不再使用独立浮起表单卡片
- 忘记密码页与重置密码页继续共用一个密码重置面板组件

### 3. 文案方向

- 以用户提供的新文案语义为准
- 允许结合当前系统场景做少量润色，以匹配页面节奏、标题长度和布局密度
- 公开密码相关页面（忘记密码 / 重置密码）底部统一仅保留“返回登录”，移除“注册新账号”链接；该规则已确认同时适用于 `ForgotPassword` 与 `ResetPassword`

## 设计目标

1. 让认证页从“脚手架式过渡界面”升级为完整产品入口页
2. 通过图标化左栏信息，明确系统提供的核心能力：设备信息、预约借还流程、通知同步
3. 让右栏表单区更轻、更直接，避免卡片层叠带来的后台模板感
4. 在不改变业务行为的前提下，统一 4 个公开路由对应的 3 类认证页面视觉语言

## 范围

### 本次包含

- 重构 `AuthLayout` 的左右分栏比例、背景层次与内容承载方式
- 调整登录、注册、忘记密码/重置密码页面的标题、说明文案与信息排布
- 在认证页引入克制的功能语义图标
- 删除忘记密码与重置密码页面中的“注册新账号”入口
- 补齐与布局、权限、业务规则相关的中文注释

### 本次不包含

- 登录、注册、重置密码接口行为变更
- 路由路径调整
- 表单校验规则调整
- 移动端适配
- 新增图片素材、插画或外部字体资源

## 信息架构

### AuthLayout

`src/layouts/AuthLayout.vue` 从“左 Hero + 右内容面板”调整为“完整双栏画布”：

- 外层仍保留公开认证背景，但减弱右侧独立面板包裹感
- 左栏保持品牌介绍职责，新增图标列表与补充说明承载能力
- 右栏改为直接承接认证页面主体，允许页面自己控制标题、表单与底部动作的节奏

### 登录页

登录页右栏聚焦单一目标：“进入设备管理工作台”。

- 标题与说明改写为更接近业务入口的语气
- 表单保持账号 + 密码两项，不增加辅助模块
- 底部继续保留“去注册”“忘记密码”两个链接

### 注册页

注册页左栏更强调“建立身份”和“账号安全要求”，右栏继续承载双列表单。

- 保留当前表单字段与两列网格
- 通过左栏补足注册含义，减少右侧说明负担
- 注册入口仍保持“注册并进入系统”的直达感

### 忘记密码 / 重置密码页

密码页统一转成“安全重置凭据”的语义。

- 左栏侧重说明恢复流程与身份校验逻辑
- 右栏保留邮箱、验证码、新密码、确认密码的操作链路
- 底部统一仅保留“返回登录”
- `ForgotPassword.vue` 与 `ResetPassword.vue` 继续通过传参区分场景文案，并通过同一组 props 控制底部辅助说明与链接集合

## 组件与文件边界

### 计划修改文件

- `src/layouts/AuthLayout.vue`
  - 调整双栏壳层结构与左栏承载内容
- `src/assets/styles/_auth-pages.scss`
  - 重写认证页公共排版与去卡片化表单基线
- `src/views/auth/Login.vue`
  - 调整登录页标题、说明与底部操作布局
- `src/views/auth/Register.vue`
  - 调整注册页标题、说明与表单区布局细节
- `src/views/auth/components/PasswordResetPanel.vue`
  - 调整密码重置页布局、辅助说明与底部链接
- `src/views/auth/ForgotPassword.vue`
  - 更新忘记密码页传入文案
- `src/views/auth/ResetPassword.vue`
  - 更新重置密码页传入文案

### 复用原则

- 不新增独立认证大组件，优先在现有结构内重组
- 忘记密码与重置密码继续共用同一表单逻辑与组件
- 样式优先收敛在 `_auth-pages.scss` 与 `AuthLayout.vue`，避免页面各自散写大量重复布局代码

## 视觉规则

### 色彩与容器

- 延续 `variables.scss` 中既有颜色变量：蓝色主按钮、青绿色链接、琥珀色眉标/辅助强调
- 页面改为“分栏 + 轻边界”结构，而不是右侧厚重白卡片
- 输入框仍保持高可读性的浅底实体感，避免因完全去容器化导致表单边界消失

### 图标策略

- 使用简单、克制、偏线性的语义图标或字符图形
- 图标仅用于辅助理解设备、流程、通知等能力，不做装饰性堆叠
- 视觉气质保持后台系统风格，不走营销页插画路线

### 排版策略

- 左栏标题保持较强品牌感与阅读节奏
- 右栏标题更短、更聚焦动作
- 描述文案控制在 1 至 2 行核心语义，避免重新堆积说明文字

## 行为与测试影响

本次不改变以下行为：

- 登录成功后的 `redirect` 优先跳转
- 注册成功后进入 `/dashboard`
- 忘记密码验证码发送后的 60 秒倒计时
- 重置密码成功后返回 `/login`

现有测试主要验证以下结构与选择器，因此实现时需要保留它们或同步更新对应测试：

- `src/views/auth/__tests__/auth-pages.spec.ts` 依赖 `.auth-panel__surface`、`.auth-panel__actions`
- 登录、注册、密码页的输入框 `name` 属性需要维持现状
- 验证码按钮的 `[data-testid="send-code-button"]` 需要保留
- `src/__tests__/App.spec.ts` 依赖 `auth-layout`、`auth-layout__hero-panel`、`auth-layout__content-panel`

## 验收标准

1. 登录页、注册页、忘记密码页、重置密码页都呈现统一的去卡片化双栏布局
2. 左栏均具备品牌/流程/图标化说明能力
3. 忘记密码与重置密码页底部仅保留“返回登录”
4. 四个公开路由涉及的核心文案完成更新，语义与已确认设计一致
5. 现有认证业务逻辑与测试约束不被破坏
6. 修改范围内补齐必要中文注释
